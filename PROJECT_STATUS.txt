KNOWN LIMITATIONS & DESIGN DECISIONS

1. No Prop Spread
   Design: Only specific props accepted (size, color, className, etc.)
   Reason: Prevents accidental override of locked stroke properties
   Result: Safer, more predictable component behavior

2. Locked Stroke Attributes
   Design: All stroke properties hardcoded in components
   Reason: Maintains visual consistency across entire icon set
   Result: Users cannot accidentally deform icons

3. No SFC for Vue
   Design: Using defineComponent + h() render function
   Reason: Simpler dependency tree, better for library distribution
   Result: No need for Vue compiler, faster rendering

4. innerHTML vs Children Rendering
   Design: SVG paths rendered via innerHTML
   Reason: Fastest way to render SVG content in React/Vue
   Result: Better performance, simpler component structure

5. One File Per Icon
   Design: 765 individual component files
   Reason: Maximum tree-shaking effectiveness
   Result: Only imported icons included in final bundle
